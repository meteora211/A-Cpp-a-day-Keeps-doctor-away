## 思路
### 哈希表
查找循环开始的listnode，第一念头是用哈希表，哈希表中第一次查询到的节点即为循环开始节点。不过需要O(N)的空间

### 双指针
瞥了一眼别人的方法，思路是：
* 先定义两个指针walker和runner从链表头开始，walker每次走一步，runner走两步。
* 如果两个指针走到NULL说明不是一个环形链表，直接返回
* 两个指针相遇时，假设walker走了k部，runner走了2k步， 链表的环形部分长为r。则有2k - k = nr => k = nr;
* 假设环形部分的第一个节点距离head为x，walker在环形部分走了y = k - x步 => x + y= nr;
* 而此时两指针相遇，runner在环形部分也走了y步，根据x + y = nr,也就是说，runner只需要再走x步就回到了环形部分开始的地方。
* 现在的问题就变成了如何让walker在runner走了x步之后也停在指针开始的地方。
* 考虑x位head到该位置的距离，那么把walker重置到head处，并与runner每次同时走一步，那么他们会在x步之后相遇，相遇的地方即为所求。
